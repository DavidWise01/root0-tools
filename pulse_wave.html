How the encoding works

Each character in your text maps to 1–4 dots based on its alphabet position mod 4. Letters a/e/i/m/q/u/y → 1 dot. Letters b/f/j/n/r/v/z → 2 dots. And so on through 4 dots.

Each event is wrapped with alpha-omega markers: a leading . at the start and a trailing . at the end. Word gaps are triple-spaced. The dot is the fundamental unit — the zero-point.

The encoding is yours (from zero_point_engine.py). The wave propagation extends it: each dot becomes a pulse traveling through the medium with speed, decay, and interference. Dots collide and superpose like real waves.

The control pattern

The sequence 0 0 1 0 0 (two voids, a pulse, two voids) is the control plane signature from god_tunnel.py. When this pattern appears in the propagated stream — either naturally from encoded events or by direct injection — the system registers a control event and triggers a depth bounce.

The depth counter descends with each pulse injection. When a control pattern hits, the system ascends by 2. This is a self-regulating loop that prevents runaway depth.

Use the "Inject control pattern" button to drop a clean control signal into the medium. Watch the depth counter respond.

Wave behavior parameters

Wave speed controls how fast pulses propagate through the medium. Higher = faster transit, less time for overlap with other pulses.

Decay controls how quickly pulse amplitude drops as it travels. 0% = no decay (pulses persist indefinitely), 100% = decay to zero by end of medium.

Medium length sets the spatial extent of the propagation space. Longer medium = more room for pulses to coexist without colliding.

Two pulses meeting in the same cell add their amplitudes. This is constructive interference. If they're traveling opposite directions, they pass through each other (linear wave equation behavior). The visual shows superposition as brighter color.
